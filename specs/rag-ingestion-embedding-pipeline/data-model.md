# Data Model for RAG Ingestion Pipeline

This document describes the data model for the documents and chunks stored in the Qdrant vector database.

## Chunk Point

Each point in the Qdrant collection represents a chunk of text from the Docusaurus site.

**Collection Name**: `docusaurus_book`

**Point Structure**:
```json
{
  "id": "string (UUID)",
  "vector": "array[float] (embedding)",
  "payload": {
    "text": "string (the chunk of text)",
    "url": "string (URL of the source page)",
    "title": "string (Title of the source page)",
    "section": "string (Section of the page, e.g. a heading)",
    "chunk_index": "integer (index of the chunk within the page)"
  }
}
```

### Fields

-   **id**: A unique identifier for the chunk. A UUID will be used.
-   **vector**: The embedding vector generated by Cohere for the `text`.
-   **payload**: A JSON object containing metadata about the chunk.
    -   **text**: The actual text content of the chunk.
    -   **url**: The URL of the page from which the chunk was extracted.
    -   **title**: The title of the page.
    -   **section**: The section of the page where the chunk was found. This can be a heading or other semantic marker.
    -   **chunk_index**: The index of the chunk within the page, for ordering.
